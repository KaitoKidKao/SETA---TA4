**3\. How to Interact with a Multi-Agent System**

This section helps evaluators understand how an AI Agent system works differently from a standard application.

**3.1. What to Expect**

| **Behavior**                     | **Explanation**                                                                                                                                   |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| Thinking time                    | Agents need time to reason and plan. Waiting 10-60 seconds is normal, not a bug.                                                                  |
| Different answers each time      | The same question may produce slightly different wording each run. Focus on whether the meaning is correct, not whether it matches word for word. |
| Agent asks you questions         | When your request is unclear, the Agent may ask for clarification instead of guessing. This is good behavior.                                     |
| Multiple agents working together | You may see the system route your request to different specialized agents. Each agent handles a different part of the task.                       |
| Visible reasoning                | Some systems show the agent thinking process. This is intentional so you can see how it reaches its answer.                                       |

**3.2. How to Write Good Requests**

| **Good Request**                                                           | **Why It Works**                                     |
| -------------------------------------------------------------------------- | ---------------------------------------------------- |
| "Summarize last quarter revenue by region and highlight any anomalies"     | Clear goal, specific scope, defined output           |
| "Compare option A and option B, then recommend which one to proceed with"  | Gives the agent a reasoning task with multiple steps |
| "Find all contracts expiring in the next 30 days and draft renewal emails" | Tests planning and tool usage across agents          |

| **Weak Request**              | **Why It Fails**               |
| ----------------------------- | ------------------------------ |
| "Do the thing"                | No context, no clear goal      |
| "Tell me everything"          | Too broad, no boundary         |
| "Yes" (with no prior context) | Agent has nothing to work with |

**3.3. Judging the Output**

| **Criteria**            | **What Counts as a Pass**                                        | **What Counts as a Fail**                                         | **Applies to**   |
| ----------------------- | ---------------------------------------------------------------- | ----------------------------------------------------------------- | ---------------- |
| Accuracy                | Core information is correct, minor phrasing differences are fine | Factually wrong, contradicts the source data                      | All              |
| Completeness            | Covers all parts of your request                                 | Misses entire sections or skips steps                             | All              |
| Reasoning quality       | Agent explains its logic or shows how it reached the answer      | Agent gives an answer with no justification or makes up reasoning | All              |
| Recovery from confusion | Agent asks for clarification or retries gracefully               | Agent halts with no explanation or produces garbage               | All              |
| Agent coordination      | Agents hand off to each other smoothly, final output is unified  | Output is fragmented, agents repeat work, or results conflict     | Multi-agent only |

**Section 2: Test Scenarios**

Each team must provide scenarios that specifically test agent behavior, not just input and output. Select from the categories below based on your system type.

**Category A: Core Functionality (minimum 2 scenarios)**

**For single-agent systems:**

| **Scenario** | **Input**                                                         | **Steps**                             | **Expected Output**                                    |
| ------------ | ----------------------------------------------------------------- | ------------------------------------- | ------------------------------------------------------ |
| Simple task  | Straightforward request within the agent's main capability        | Number each step, include screenshots | Describe the correct result                            |
| Complex task | Request that requires planning, reasoning, or multiple tool calls | Number each step, include screenshots | Describe how the agent breaks down and solves the task |

**For multi-agent systems** - include the above plus:

| **Scenario**     | **Input**                                 | **Steps**                                    | **Expected Output**                                          |
| ---------------- | ----------------------------------------- | -------------------------------------------- | ------------------------------------------------------------ |
| Multi-agent task | Request that requires more than one agent | Number each step; note which agents activate | Describe how agents coordinate and produce a combined result |

**Category B: Agent Intelligence (minimum 2 scenarios)**

| **Scenario**               | **What It Tests**                                 | **Input**                                                       | **What to Observe**                                                          | **Applies to**   |
| -------------------------- | ------------------------------------------------- | --------------------------------------------------------------- | ---------------------------------------------------------------------------- | ---------------- |
| Memory and context         | Does the agent remember what you said earlier     | Ask a follow-up question that depends on the previous answer    | Agent uses prior context without asking again                                | All              |
| Autonomy vs clarification  | When does the agent decide on its own vs ask you  | Give a vague request with multiple valid interpretations        | Agent either picks the best option and explains why, or asks you to clarify  | All              |
| Tool selection             | Does the agent pick the right tool for the job    | Ask the same question in two different ways                     | Agent may use different tools but reaches the same correct answer            | All              |
| Planning and decomposition | Can the agent break a big task into smaller steps | Give a task with 3 or more sub-tasks                            | Agent identifies the steps, executes them in order, and combines the results | All              |
| Agent handoff              | Do agents pass work to each other correctly       | Give a task that spans multiple agent specialties               | Work flows from one agent to the next without losing information             | Multi-agent only |
| Conflict resolution        | When agents disagree, how is it resolved          | Give a request where agents might produce contradicting outputs | System resolves or surfaces the disagreement clearly                         | Multi-agent only |

**Category C: Error and Edge Cases (minimum 1 scenario)**

| **Scenario**             | **Input**                                         | **What to Observe**                                                              | **Applies to**   |
| ------------------------ | ------------------------------------------------- | -------------------------------------------------------------------------------- | ---------------- |
| Conflicting instructions | Give contradictory requirements                   | Agent flags the conflict instead of producing wrong output                       | All              |
| Beyond-scope request     | Ask for something the system cannot do            | Agent explains its limits clearly rather than hallucinating                      | All              |
| Ambiguous input          | Give an incomplete or vague request               | Agent asks for clarification rather than assuming                                | All              |
| Partial failure          | A task where one agent succeeds but another fails | System reports what worked and what did not; does not pretend everything is fine | Multi-agent only |

Single-agent systems: Minimum 4 scenarios total, maximum 6.

Multi-agent systems: Minimum 5 scenarios total, maximum 8.

**Section 4: Known Limitations**

Be honest about what the system cannot do. Evaluators will appreciate transparency over surprises.

| **Category**               | **Description**                                        | **Applies to** |
| -------------------------- | ------------------------------------------------------ | -------------- |
| Supported languages        | e.g. Vietnamese and English only                       | All            |
| Input constraints          | e.g. Maximum file size, supported formats              | All            |
| Response time              | e.g. 10-30 seconds per request                         | Single agent   |
| Response time              | e.g. 30-90 seconds when agents coordinate              | Multi-agent    |
| Conversation memory limit  | e.g. Remembers last 10 exchanges                       | All            |
| What the agent cannot do   | e.g. Cannot access external websites                   | Single agent   |
| Number of agents           | e.g. System uses 3 agents: Researcher, Analyst, Writer | Multi-agent    |
| What each agent cannot do  | e.g. Researcher cannot access internal databases       | Multi-agent    |
| Features not yet available | e.g. Voice input, real-time collaboration              | All            |

**Section 5: Quick Reference**

| **Do**                                                       | **Do Not**                                        |
| ------------------------------------------------------------ | ------------------------------------------------- |
| Write requests as if you are talking to a smart colleague    | Use single-word commands with no context          |
| Try complex tasks that require multiple steps                | Expect the same exact wording every time you ask  |
| Test follow-up questions to check if it remembers context    | Enter personal or sensitive information           |
| Give vague requests on purpose to see how the agent responds | Assume a slow response means the system is broken |
| Read the thinking or reasoning trace if it is visible        | Ignore error messages from the system             |
| Watch how the agent selects tools or plans its steps         | Rush the system by clicking repeatedly            |
| Watch how agents hand off work to each other (multi-agent)   | Expect all agents to respond at the same speed    |
