**5.2 Mandatory 8-Slide Structure**

| **1** | **Opening**                                                                                                                                                                                                                                                                                                         | **0:20** |
| ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- |
|       | **Team name: \[**Team Name\]<br><br>**Project name: \[**Project Name\]<br><br>**Elevator pitch:** For \[target audience\], we built an AI Agent that \[function\] to solve \[problem\]<br><br>**Team members:** Name and role of each member<br><br>**Objective:** Make a strong first impression within 30 seconds |          |

| **2** | **Problem Statement**                                                                                                                                                                                                                                                                                                                                                         | **1:20** |
| ----- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- |
|       | **Business context:** Brief overview of the industry or domain<br><br>**Core problem:** Specific pain point with supporting data if available<br><br>**Current approach:** Why the existing solution is insufficient<br><br>**Affected stakeholders:** Who is impacted by this problem<br><br>**Objective:** Ensure the judges clearly understand why this solution is needed |          |

| **3** | **Solution Overview**                                                                                                                                                                                                                                                                                                                                                 | **1:20** |
| ----- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- |
|       | **Overall solution:** How the AI Agent addresses the problem<br><br>**Why an Agent:** Why an Agent is more suitable than a standard chatbot or automation<br><br>**Core value:** Primary benefit delivered to users<br><br>**User journey:** Summary of the end-to-end user experience<br><br>**Objective:** Demonstrate the practical value of the proposed solution |          |

| **4** | **Architecture & Technology Stack**                                                                                                                                                                                                                                                                                                                                                                                                                                                     | **2:00** |
| ----- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- |
|       | **Architecture diagram:** System overview - visual diagram is mandatory<br><br>**LLM selection:** Model name and rationale for choosing it<br><br>**Agent framework:** Framework used to build the Agent<br><br>**Integrated tools:** APIs and external services connected<br><br>**Data layer:** Database, vector database, data pipeline<br><br>**Deployment:** Hosting environment and infrastructure<br><br>**Objective:** Demonstrate technical depth and system design capability |          |

| **5** | **Agentic Design**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             | **2:00** |
| ----- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- |
|       | **Reasoning:** How the Agent reasons through tasks<br><br>**Planning:** How the Agent decomposes and sequences subtasks<br><br>**Memory:** Short-term and long-term context retention mechanisms<br><br>**Tool usage:** Criteria the Agent uses to select tools<br><br>**Autonomy level:** When the Agent acts independently vs. requires human input<br><br>**Multi-Agent coordination:** If applicable, how multiple Agents collaborate<br><br>**Objective:** Prove that this is a genuine Agent system, not merely a language model wrapper |          |

|     | **6\. PART 2 - LIVE DEMO GUIDELINES** |
| --- | ------------------------------------- |

**6.1 Demo Structure Overview**

| **Phase** | **Content**                     | **Duration** |
| --------- | ------------------------------- | ------------ |
| 1         | Context setup                   | 1 minute     |
| 2         | Core flow demonstration         | 3 minutes    |
| 3         | Advanced scenario demonstration | 3 minutes    |
| 4         | Error and edge case handling    | 2 minutes    |
| 5         | Open demo per judge request     | 1 minutes    |

**6.2 Detailed Phase Breakdown**

| **Phase 1 Context** Setup · 1 minute | |
| --- | | --- |
| **Demo environment** | Local, cloud, or staging |
| **Scenarios covered** | Outline what will be demonstrated |
| **Demo data** | Real data or synthetic data |

| **Phase 2 Core** Flow Demonstration · 3 minutes | |
| --- | | --- |
| **Step 1** | Provide input or trigger the Agent |
| **Step 2** | Agent receives and analyzes the task |
| **Step 3** | Agent selects and invokes tools to gather information |
| **Step 4** | Agent synthesizes and returns results |
| **Step 5** | Compare output against expected results |

| **Phase 3 Advanced** Scenarios · 3 minutes | |
| --- | | --- |
| **Multi-step reasoning** | Complex task requiring sequential reasoning |
| **Multi-Agent collaboration** | Multiple Agents working together (if applicable) |
| **Dynamic tool selection** | Agent adapts tool choice based on context |
| **Context retention** | Agent leverages information from prior interactions |
| **Adaptive behavior** | Agent changes strategy when conditions shift |

| **Phase 4 Error** & Edge Case Handling · 2 minutes | |
| --- | | --- |
| **Invalid input** | Wrong format, missing data, different language |
| **API failure** | External service unresponsive |
| **Ambiguous request** | Agent asks for clarification or infers intent |
| **Hallucination control** | Mechanism for self-checking output |
| **Human-in-the-loop** | Conditions that trigger a pause for user confirmation |

| **Phase 5 Open** Demo per Judge Request · 1 minutes | |
| --- | | --- |
| **Additional requests** | Judges may ask for new scenarios on the spot |
| **Bonus features** | Teams may demonstrate extra capabilities if time permits |

|     | **7\. BACKUP DEMO POLICY** |
| --- | -------------------------- |

| **Situation**               | **Resolution**                                           |
| --------------------------- | -------------------------------------------------------- |
| Live demo succeeds          | Scored under standard criteria                           |
| Live demo fails mid-process | One restart allowed within the allotted time             |
| Live demo cannot continue   | Switch to backup video; judges determine score deduction |
| No demo available           | Demo section scored as zero                              |

**⚠ Mandatory Backup Requirement**

Each team must submit a backup demo video of 3-5 minutes before the competition date.
